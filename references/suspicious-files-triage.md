# Reference: suspicious files triage

Процесс и решения по эталону Amanita зафиксированы в `docs/architecture-extraction/12-suspicious-or-temporary-items.md`.

## Не поднимать в reusable bootstrap без отдельного решения

- Локальные бэкапы каталогов скриптов, zip-архивы в корне.
- Сторонние референс-проекты (не путать с каноническим `wallet/`).
- Личные/мета каталоги вне продукта.
- Одноразовые датасеты, `gasReporterOutput`, scratch-файлы без имени.

## Безопасно включать в описание архитектуры эталона, но не копировать файлами

- `keys/` — только README с инструкцией генерации.
- Политика секретов: uploader ↔ node ↔ GPT — три разных роли секретов.

## Паттерн «подозрительно, но системно»

Долгие валидаторы (`scripts/validators/*`) — **системные** для качества данных перед on-chain; в bootstrap — **опциональная** фаза, не обязательный минимум.
