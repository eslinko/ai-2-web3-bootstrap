# keys/

Каталог для **локально сгенерированных** ключей. В шаблоне репозитория здесь только этот README — **никаких** значений ключей из донорских проектов не копировать.

## Связь node ↔ arweave-uploader

Ключи используются для доверенной связи между **web2-нодой** (`node/`) и **`arweave-uploader`**: секреты в `.env` и материалы для подписи data items должны быть **согласованы** с вашей политикой деплоя.

## Что сгенерировать (ориентиры)

1. **Arweave (JWK)** — сгенерировать новый кошелёк Arweave и сохранить JWK в файл внутри `keys/` (имя зафиксировать в `.env` uploader). Не публиковать в git.
2. **Секреты сервис-сервис** — сгенерировать криптостойкие строки для `EDGE_TO_BACKEND_SECRET` / `UPLOADER_TO_BACKEND_SECRET` / `GPT_ACTIONS_BEARER_SECRET` и задать одинаково там, где требует контракт (см. `templates/env/shared-vars-catalog.md`).
3. **EVM** — для деплоя и продавца использовать отдельные ключи из безопасного хранилища; в репозиторий не класть.

Добавьте в **корневой `.gitignore`**: `keys/*.json`, `keys/*.pem` и любые маски ваших файлов ключей.
