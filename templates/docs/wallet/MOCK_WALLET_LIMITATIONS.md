# Mock wallet — ограничения

Этот документ предотвращает путаницу между **временным signing layer** и **финальным wallet product**.

## Strictly temporary (типично для mock-runner паттерна)

- Один процесс, один пользовательский identity для E2E.
- Секреты в `.env` рядом с клиентом — только dev; не production custody.
- Зависимости от локальных путей (фикстуры, маркеры готовности для shell-скриптов) — не контракт для продакшена.
- Отсутствие реального UX (согласие пользователя, выбор сети, аппрувы).

## Reusable как паттерн

- Очередь подписей на **сервере** (нода), тонкий **клиент подписи**.
- Явное разделение Arweave vs EVM контуров.
- Документированные env-группы и роли переменных.

## Что должно заменить mock позже

- Внешний wallet (браузерное расширение, WalletConnect, аппаратный кошелёк) или отдельный подписывающий сервис с политикой секретов.
- Secret management (Vault и т.д.) вместо flat `.env`.
- Политики multi-user / организаций — только если появятся в продукте.

## Риски при ошибочном использовании

- Принятие mock за дизайн финального wallet → неверные ожидания по безопасности и UX.
- Смешивание ключей Arweave и EVM в одном «мастер-ключе» — антипаттерн для этого контура.
