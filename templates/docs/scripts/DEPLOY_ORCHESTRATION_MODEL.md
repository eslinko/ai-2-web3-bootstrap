# Deploy orchestration model

## Варианты (решение `scripts.deploy_pattern`)

### `router_based`

- Единый entry (например `deploy_router.js`) выбирает операцию по аргументу/переменной.
- **Границы:** router отвечает за **разбор аргументов, подключение config, вызов зарегистрированных handlers**. Конкретная логика контрактов живёт в **отдельных модулях**, не в одном гигантском файле.

### `simple_named_scripts`

- Несколько независимых Node/bash скриптов без общего роутера.
- Подходит для небольших репо.

### `defined_later`

- Плейсхолдер: оркестрация добавляется после интервью вручную; bootstrap даёт только docs/reference.

## Паттерн из эталона

Эталонный `deploy_full.js` + `lib/actions` — **образец оркестрации**, не обязательный bundle. Стартер: минимальный **registry** + заглушки handlers.
