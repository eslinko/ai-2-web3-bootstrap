# Validation model (scripts)

## Phase-based (решение `scripts.validation_pattern = phase_based`)

- Валидаторы привязаны к **фазам** пайплайна (например перед `chain`, после загрузки CSV).
- Один **минимальный файл-шаблон** в scaffold показывает структуру: экспорт функции `validate(context)` или набор фаз.

## Что переносить из эталона

- **Форма:** фаза, входные данные, отчёт об ошибках, graceful fail vs throw.
- Не переносить **десятки файлов** и доменную логику эталона целиком.

## Когда достаточно minimal / none

- **`minimal_checks`** — одна проверка env/путей перед запуском.
- **`none`** — валидация только на стороне node/contracts; scripts ограничены smoke.
